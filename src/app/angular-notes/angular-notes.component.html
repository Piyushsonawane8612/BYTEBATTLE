Angular


Angular 7 is an open source JavaScript framework for building web applications and apps in JavaScript, html, and Typescript which is a superset of JavaScript. Angular provides built-in features for animation, http service, and materials which in turn have features such as auto-complete, navigation, toolbar, menus, etc. The code is written in Typescript, which compiles to JavaScript and displays the same in the browser.

Angular7 - Environment Setup

In this chapter, we will discuss the Environment Setup required for Angular 7. To install Angular 7, we require the following −

  • Nodejs
  • Npm
  • Angular CLI
  • IDE for writing your code

Node.js and npm Installation:

  • Install Node.js and npm by downloading the installer from the official website: Node.js.

Angular CLI Installation:

  • Open a terminal or command prompt and run the following command to install Angular CLI globally:

npm install -g @angular/cli


IDE for Writing Code:

  • Choose an Integrated Development Environment (IDE) for writing your Angular code. Some popular choices include:
    • Visual Studio Code
    • WebStorm

Create Angular Project:

  • Open a terminal or command prompt and navigate to the desired directory for your project.
  • Run the following command to create a new Angular project:

    • ng new your-project-name

  • Follow the prompts to set up your project. You can choose additional features like routing and stylesheets during the setup.

Navigate to Project Directory:

  • Change into the project directory:

    • cd your-project-name

Run the Angular App:

  • Start the development server with the following command:

    • ng serve

  • Open your browser and navigate to http://localhost:4200/ to see your Angular app.

The src/folder is the main folder, which internally has a different file structure.

Folder Structure:

  • src/ is the main folder.
  • app/ contains default files installed by Angular CLI.
  • app.module.ts imports essential libraries and configures the main module.
  • app-routing.module.ts handles routing for the project.

Files in app/:

  • app.component.css: Contains CSS for styling.
  • app.component.html: Default HTML content with placeholder links.
  • app.component.spec.ts: Auto-generated unit test file.
  • app.component.ts: Defines the main component class with a title.
  • app-routing.module.ts: Manages routing for the app.

Additional Folders:

  • assets/: For saving images and JS files.
  • environment/: Contains environment configuration files.

Additional Files:

  • favicon.ico: Website icon.
  • index.html: Main HTML file with <app-root></app-root> selector.
  • main.ts: Starting point for the project, bootstraps the main module.
  • 'polyfill.ts', 'styles.css', 'test.ts', 'tsconfig.app.json', 'tsconfig.spec.json',
  • typings.d.ts: Various configuration and utility files.

Execution Flow:

  • Project starts from 'main.ts'.
  • Imports necessary modules and environment settings.
  • Bootstraps the main module '(AppModule)', triggering the display of the app

Angular7 - Modules

Module in Angular refers to a place where you can group the components, directives, pipes, and services, which are related to the application.

In case you are developing a website, the header, footer, left, center and the right section become part of a module.

To define module, we can use the NgModule. When you create a new project using the Angular –cli command, the ngmodule is created in the app.module.ts file by default

The NgModule needs to be imported as follows − import { NgModule } from '@angular/core';

Declaration

It is an array of components created. If any new component gets created, it will be imported first and the reference will be included in declarations as shown below −

declarations: [ 
AppComponent,  
NewCmpComponent 
]

Import

It is an array of modules required to be used in the application. It can also be used by the components in the Declaration array. For example, right now in the @NgModule, we see the Browser Module imported. In case your application needs forms, you can include the module with the below code −

 import { FormsModule } from '@angular/forms';
The import in the @NgModule will be like the following −
imports: [ 
BrowserModule, 
FormsModule 
]

Providers

This will include the services created.

Bootstrap

This includes the main app component for starting the execution.



NgModule Overview:

Purpose of NgModule:

Angular's way to group related code blocks (components, directives, pipes, services). Describes how the application components fit together

Similarity to Namespace (C#)Resembles a namespace in C#. Organizes dependencies for both the compiler and dependency injection. Enhances code organization and readability.

Modules can be lazily loaded, improving application startup speed. Useful when modules are not needed immediately on startup

Every Angular application must have a root module (e.g., AppModule). The app is launched by bootstrapping the root module

Structure of NgModule:

Defined by the '@NgModule' decorator. Includes key pieces in the metadata object:

    Declarations Array:
  • Contains components, pipes, and directives specific to the module.
    Imports Array
  • Imports external and internal modules.
  • Examples: CommonModule, custom routing modules.
    Providers Array:
  • Includes internal and external services.
  • Globally scoped, no need for export.
  • Example: HttpClientModule.
    Bootstrap Array:
  • Specifies the main app component for application entry.
  • Components listed here are created during bootstrapping.

Multiple Modules:

NgModules organize applications into cohesive groups. Feature modules complement the root module for focused functionality. Helps separate concerns like routing, global services, and UI components.
